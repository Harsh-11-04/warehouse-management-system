# Jattkart POS System

A modern, offline-first MERN stack Point of Sale (POS) system wrapped inside a blazing fast Windows Desktop Application using **Electron**. Built specifically for retail/warehouse environments.

## Architecture
- **Frontend**: React (Vite) styled with Tailwind CSS / Custom CSS.
- **Backend Server**: Node.js + Express (auto-started unconditionally inside the Desktop App).
- **Desktop Wrapper**: Electron (packages frontend and backend into a standalone `.exe`).
- **Database**: MongoDB Atlas (Cloud) OR Local MongoDB with automatic background syncing (Sync Worker).

---

## 🚀 Getting Started (Development Mode)

If you are a developer looking to write code or test new features, run the development environment.

### Prerequisites:
- Node.js installed on your machine.
- MongoDB connection string configured in `backend/.env`.

### 1. Install Dependencies
Run this in the root directory (it automatically installs both frontend and backend dependencies):
```bash
npm install
```

### 2. Start Electron Dev Mode
Run the Electron App in development mode. This will build your Vite frontend and immediately launch the Electron desktop app locally with hot-reloading active.
```bash
npm run electron:dev
```
*Note: The backend runs natively inside the Electron App on port `4000` via a background worker process.*

## 📦 Building the Production App (Windows .exe)

When you are ready to distribute the fully functioning App to the Cashier PCs, you package it into a single Portable `.exe` file.

### 1. Build the Desktop App
```bash
npm run electron:build
```
*Note: This process compiles the React frontend into static files, packages the Express backend, and binds them together using `electron-builder`.*

### 2. Locate your `.exe`
Once the build is complete (approx. 2 minutes), you will find your portable executable here:
```
dist-electron/Jattkart POS 1.0.0.exe
```
This is a zero-install **Portable Executable**. You can put this on a USB drive, move it to any Windows PC, double-click it, and the entire POS will launch instantly!

## ⚙️ Configuration & Environment Variables

The Desktop App relies on the `.env` file located in the `backend/` directory. **Ensure your `.env` is correctly configured BEFORE building the `.exe`.**

### Connecting to the Database
If your `backend/.env` has `MONGO_URI` pointing to **MongoDB Atlas**, the Cashier PC will act as an "Always Online" cloud checkout terminal. 

If you want the Cashier PC to be "Offline-First", you must point `MONGO_URI` to `mongodb://127.0.0.1:27017/jattkart` and enable your background Sync Worker.

```env
# backend/.env 
PORT=5000
NODE_ENV=development

# Cloud Database
MONGO_URI=mongodb+srv://<user>:<password>@cluster.mongodb.net/?appName=Jaat-Canteen
```

## 🛠️ Project Structure

```
warehouse-management-system/
├── backend/               # Express.js REST API & MongoDB Models
│   ├── src/               # Controllers, Routes, Services
│   ├── workers/           # Offline-to-Cloud Sync Worker (sync.worker.js)
│   └── server.js          # Entry point for backend
├── frontend/              # React POS User Interface
│   ├── src/               # React Components & Pages
│   └── vite.config.ts     # Vite Config (base: './' for Electron)
├── electron/              # Desktop App Wrapper Logic
│   ├── main.js            # Electron Main Process (Spawns backend/frontend)
│   ├── preload.js         # Secure bridge for IPC context
│   └── splash.html        # Loading Screen 
├── dist-electron/         # (Generated) Where the final .exe lives
└── package.json           # Root scripts & electron-builder configuration
```

## Maintenance & Logs
If the Desktop app crashes or behaves unexpectedly, you can read the internal process logs generated by the packaged `.exe` at:
- `%APPDATA%\jattkart-pos\logs\electron.log`
- `%APPDATA%\jattkart-pos\logs\backend-stdout.log`
